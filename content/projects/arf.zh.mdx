# ARF / Ares Robotics Framework

## 概述

ARF（Ares Robotics Framework）是一套采用云边协同架构设计的通用机器人操作系统。该框架通过提供标准化的通用控制框架与生态基础，解决底层硬件与算法的碎片化问题。

## 架构

ARF 采用分层架构，实现关注点分离，并支持在分布式机器人系统中进行可组合的决策。

### 边缘侧架构

边缘侧由六个各司其职的层级组成，每一层都有明确的职责：

**L0：HAL（硬件抽象层）**
- 基于 gRPC 的驱动即服务架构
- 通过 Rust/C++ 实现崩溃隔离
- 为上层提供与硬件无关的接口

**L1：RTS（实时调度）**
- 无锁 SPSC（单生产者单消费者）队列实现
- 在 PREEMPT_RT 内核上以 1 kHz 实时调度频率运行
- 抖动低于 100 微秒，确保控制的确定性
- 为时间敏感的机器人操作提供时序可预测性保证

**L2：DMS（数据管理系统）**
- 基于 Redis/ZeroMQ 的发布-订阅架构
- 环形缓冲区的事件触发式记录器
- 采用 Parquet 格式进行云端同步
- 实现高效的数据流转与历史回放

**L3：ACR（算法容器运行时）**
- 基于 Go 运行时，承载 Python 容器
- DAG（有向无环图）执行模型
- 容器间通信采用零拷贝共享内存
- 将算法逻辑与系统基础设施隔离

**L4：DIL（决策智能层）**
- 基于 Python/PyTorch 实现
- 用于状态估计与预测的世界模型
- 采用 py_trees 行为树进行分层任务分解
- 具备最高优先级、拥有覆盖权限的安全监控器
- 自主-遥操作模式管理器
- 用于高层意图解读的 LLM 任务规划器

**L5：App（应用层）**
- FastAPI 后端 + React 前端
- 面向用户的系统配置与监控界面
- 高层任务定义与执行管理

### 云侧组件

云侧为大规模机器人集群管理与持续改进提供基础设施：

- **Data Hub**：基于 Iceberg 的湖仓，用于统一的数据存储与查询
- **Converter**：基于 Argo DAG 的数据转换流水线
- **Training**：基于 Kubeflow/Ray 的分布式训练，支持 Sim2Real 迁移与联邦学习
- **Isaac Sim**：集成 NVIDIA Isaac Sim，用于仿真与合成数据生成
- **Evaluation**：自动化的模型与算法性能评估
- **Marketplace**：基于 Harbor 的容器镜像仓库，用于算法共享与分发
- **Fleet Management**：基于 KubeEdge 的编排，配合 OTA A/B 部署实现零停机更新

### DIL 三层可组合决策架构

决策智能层采用三层可组合架构：

**L1：原子算法能力**
- 以 ACR 容器形式封装的单个算法能力
- 无状态、可复用的计算单元
- 示例：目标检测、路径规划、运动控制

**L2：可组合决策组件**
- 用于任务编排的行为树管理器
- 视觉搜索与目标跟踪
- 具备实时约束校验的安全监控器
- 组合 L1 能力的可复用决策原语

**L3：完整决策实例**
- 由开发者组装的完整决策系统
- 面向具体应用的行为组合
- 示例：仓储 AMR 导航、操作任务执行

这种可组合架构使开发者能够通过组合经过测试、可复用的组件来构建复杂的机器人行为，而无需从零编写单体式控制系统。

## 技术规格

### 实时性能

- **调度频率**：1 kHz（1 毫秒周期时间）
- **抖动**：< 100 μs（微秒）
- **内核**：PREEMPT_RT 实时 Linux 补丁
- **队列架构**：无锁 SPSC，实现零竞争的数据流转

### 通信架构

- **层间通信**：采用 gRPC 进行强类型的服务调用
- **数据流**：采用 ZeroMQ 发布-订阅，处理高吞吐的传感器数据
- **云端同步**：采用 Parquet 列式格式，实现高效存储与查询

### 决策智能栈

- **框架**：采用 PyTorch 构建神经网络模型
- **行为树**：采用 py_trees 库进行分层任务分解
- **世界模型**：用于基于模型控制的状态估计与预测
- **安全**：最高优先级监控器，对所有动作拥有否决权
- **模式管理**：在自主、遥操作与混合模式之间无缝切换
- **任务规划**：集成 LLM 进行自然语言任务解读

## 技术栈

- **RT 内核**：PREEMPT_RT Linux
- **HAL**：Rust / C++，配合 gRPC 服务层
- **RTS**：无锁 SPSC 队列，1 kHz 调度
- **DMS**：Redis、ZeroMQ、Parquet
- **ACR**：Go 运行时、Python 容器、DAG 执行、零拷贝共享内存
- **DIL**：Python、PyTorch、py_trees、LLM 集成
- **App**：FastAPI（后端）、React（前端）
- **Cloud**：KubeEdge、Kubeflow、Ray、Argo、Harbor、Isaac Sim、Iceberg

## 任务序列与数据飞轮

ARF 实现了一套完整的数据飞轮，持续提升机器人性能：

1. **任务定义**：用户通过 App 层或 LLM 自然语言接口定义高层任务
2. **任务分解**：DIL 行为树将任务拆解为可执行的原语
3. **执行**：RTS 以 1 kHz 调度算法容器，并提供实时保证
4. **数据采集**：DMS 将所有传感器数据、动作与结果记录到环形缓冲区
5. **云端同步**：以事件触发方式，将数据以 Parquet 格式上传至 Iceberg 数据中心
6. **训练**：在云端使用真实世界数据 + 仿真数据（Sim2Real）进行分布式训练
7. **评估**：针对基准场景进行自动化性能评估
8. **部署**：通过 OTA A/B 部署将改进后的模型推送至集群，并具备回滚能力
9. **持续改进**：更新后的算法回流至第 3 步，形成闭环

该飞轮使基于 ARF 的机器人能够从部署经验中持续学习，并随时间推移不断提升自主能力。

## 价值主张

ARF 解决了机器人领域根本性的碎片化问题：每个机器人项目都在从零重建同一套基础设施栈（硬件抽象、实时控制、数据管理、决策逻辑）。这种碎片化浪费工程投入、阻碍算法复用，并拖慢创新。

ARF 提供：

- **标准化基础设施**：以经过验证、可用于生产的技术栈，消除各项目重复的基础设施开发
- **硬件无关性**：HAL 抽象使算法能够在不同机器人平台间移植
- **可组合智能**：DIL 三层架构支持通过经过测试、可复用的组件构建复杂行为
- **实时保证**：PREEMPT_RT + 1 kHz 调度 + 亚 100μs 抖动，为安全关键应用提供确定性控制
- **生态基础**：Marketplace + 云端训练 + 集群管理，构建算法提供方与机器人运营方共同繁荣的生态
- **持续改进**：内置的数据飞轮将每一次机器人部署都转化为一次训练机会

通过标准化控制框架，ARF 让机器人工程师能够专注于差异化的算法创新，而非基础设施的底层搭建。其结果是更快的开发周期、更高质量的算法，以及一条通往持续自主改进的路径。

## 路线图

当前的 ARF 实现聚焦于边缘侧核心层（从 HAL 到 DIL），并在生产环境部署中展示了在 PREEMPT_RT 系统上实现 1 kHz 控制与亚 100μs 抖动的能力。云侧组件正在积极开发中，其中 Data Hub 与 Fleet Management 已进入 beta 测试阶段。

未来开发的优先事项包括：

- 扩展硬件驱动库（HAL）
- 多机器人协同原语（DIL）
- 跨机器人集群的联邦学习（Cloud Training）
- 结合域随机化的增强型 Sim2Real 迁移（Isaac Sim 集成）
- 面向第三方算法贡献的开放 Marketplace
